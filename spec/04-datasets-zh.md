## 1. 模拟器设计

共享模拟器核心（`src/lhmsb/sim/core.py`，任务 7）通过一个与任务族无关的流水线生成 episode。每个 episode 是一个固定的、从种子派生的世界事件和对齐探针计划——在 v1 中，智能体不会改变世界（交互式世界推迟到 v2）。

### 1.1 核心概念

**Episode（回合）**：跨多个会话的独立任务。由 `episode_id` 标识，标记为 `family` 和 `seed`。包含一个有序的 `WorldEvent` 列表和一个对齐的 `Probe` 列表。

**WorldEvent（世界事件）**：在特定步骤对外生证据世界的更改：
- `inject`：新事实进入有效集。
- `change`：现有事实被精炼或替换。
- `retract`：先前有效的事实被撤回（不再为真）。

**Probe（探针）**：智能体在特定步骤必须回答或执行的问题或任务。携带：
- `probe_id`：Episode 内唯一。
- `step`：探针在哪个步骤发放。
- `kind`：factual（确定性黄金答案）、synthesis（开放式，判定器辅助）或 behavioral（测试约束遵守情况）。
- `gold`：给定探针步骤世界状态的正确答案（已揭示减去已撤回的事实）。
- `cross_session`：探针是否需要来自其自身会话之前的会话的事实。

**World state（世界状态）**：步骤 `t` 的当前有效事实集，定义为 `{截至 t 注入的事实} \ {截至 t 撤回的事实}`。探针黄金答案从该状态派生。

**world_event_hash（世界事件哈希）**：基于有序的外生事件和探针计划的稳定哈希。对于相同 `(episode_id, seed)` 在所有记忆条件间相同，强制反事实不变量。

### 1.2 渲染

模拟器通过 LLM（`SurfaceRenderer`）将结构化世界事件和探针渲染为自然语言文本，产生智能体阅读的表面文本。渲染文本按 `(episode_id, seed, step)` 键进行**冻结缓存**，因此渲染是可复现的，其成本被排除在系统成本核算之外。`validate_render(episode)` 守卫断言渲染文本不与该步骤的真实世界状态相矛盾或泄露超出其范围的事实。

### 1.3 检查器协议

每个任务族模块实现一个 `Checker`，程序化地根据探针黄金答案评析智能体的答案。检查器返回一个 `[0,1]` 分数以及结构化元数据（使用了哪些事实、漂移标记、按标准通过/失败）。只有程序化无法检查的探针才被推迟到稀疏判定器——判定器比例有上限并单独报告。

## 2. 任务族范围上限（防爆炸）

这些上限防止基准在执行 episode 期间需要真实世界基础设施、网络访问或无界计算。两个任务族在数据集生成后都必须能够完全离线运行。

### 2.1 软件开发族

**概念**：智能体跨多个会话接收不断演进的软件规格。需求和约定随时间被注入、变更和撤回。探针测试智能体是否记住并正确应用当前的需求集——而非陈旧的需求。

**范围上限**：

| 约束 | 限制 |
|------|------|
| 包大小 | ≤ 6 个文件（`.py` + 测试文件） |
| 文件长度 | ≤ 每文件 200 行 |
| 依赖 | 仅 Python 标准库（episode 期间无 `pip install`） |
| 网络访问 | **无**——沙箱子进程，`network=none` |
| 测试框架 | `pytest`（确定性，离线） |
| 代码执行 | 沙箱子进程，时间（≤ 30s）、内存（≤ 256 MB）和输出（≤ 10 KB）限制 |
| Episode 会话 | 2–5 个会话，模拟数天/数周的规格演进 |
| 每 episode 需求 | 5–15 个总事件（注入/变更/撤回混合） |

**探针类型**：
- **实现**：编写/修改代码以满足当前规格。
- **约定遵守**：应用仍然活跃的命名/风格/API 约定。
- **弃用意识**：在旧 API 被撤回后使用新 API。
- **测试驱动**：代码必须通过当前 `R_t` 的隐藏 `T_t`（pytest 套件）。

**评分**：`SoftwareChecker` 在沙箱子进程中运行 `T_t` 并解析 pytest 结果。静态检查验证约定遵守。漂移检测检查陈旧 API 使用和约束违规。

### 2.2 研究项目族

**概念**：智能体在合成证据世界上进行自主调查。发现被逐步注入，有些被精炼，有些被撤回（被证伪）。探针测试智能体是否正确综合当前证据，且不引用被撤回的发现。

**范围上限**：

| 约束 | 限制 |
|------|------|
| 实体名称 | 仅合成——从种子名称列表生成（例如"Project Chimera"、"Entity-42"、"Study Gamma-7"）。无真实论文标题、作者、DOI。 |
| 证据 ID | 生成的 `fact_id`（例如 `ev-001`、`ev-002`）。所有探针黄金答案映射到已知合成事实。 |
| 事实数量 | 每 episode 15–40 个事实 |
| 撤回率 | 20%–40% 的注入事实随后被撤回或替代 |
| Episode 会话 | 3–6 个会话，模拟数月的调查 |
| 证据图 | 冻结 DAG——事实之间的依赖关系从种子生成并固定。被撤回的父级级联影响依赖事实。 |
| 智能体动作 | 不改变证据图（v1 固定世界）。 |

**探针类型**：
- **事实回忆**："Study X 对 Y 得出了什么结论？"（黄金答案 = 当前未被撤回的综合）。
- **更新正确性**："基于最新证据，当前对 Z 的理解是什么？"（不得引用被撤回的发现）。
- **综合**："总结主题 T 上的证据现状。"开放式——推迟到稀疏判定器及评分标准。
- **目标遵守**："这条调查路线是否仍然与最初的研究问题一致？"（约束检查）。

**评分**：`ResearchChecker` 将答案中的每个事实声明映射到已知合成 `fact_id`。事实/更新探针为确定性；综合探针推迟到判定器。漂移检测检查陈旧事实引用和目标约束违规。

**泄露防护**：`lint_no_real_entities()` 扫描生成文本中的已知真实世界标识符（论文标题、作者姓名、DOI 模式）。任何匹配在生成时被拒绝。

## 3. 可复现性

### 3.1 冻结 + 校验和

Pilot 数据集生成一次，验证并冻结到版本化的文件，位于 `datasets/<family>_pilot/`。每个冻结集包含：

- `episodes.jsonl`：每个 episode 一个 JSON 对象（事件、探针、黄金答案、world_event_hash、episode_hash）。
- `rendered/`：按步骤渲染的文本，按 `(episode_id, seed, step)` 键。
- `MANIFEST.json`：生成器版本、git SHA、配置哈希、种子、规模参数、每文件 SHA-256 校验和、生成时间戳。
- `dataset_card.md`：可读摘要（见第 4 节）。

`verify` 命令重新计算校验和并断言它们与清单匹配。被篡改的文件（一个字节翻转）导致 `verify` 以非零退出并附带不匹配报告。

### 3.2 基于种子的重新生成

`regen-check` 命令从存储的种子重新生成 episode，并断言**完全相同**的 `world_event_hash` 和 `episode_hash` 值。这证明生成配方是确定性的，可仅从种子复现——即使没有冻结文件。

种子、规模参数和生成器版本都存储在清单中。智能体和判定器模型不在数据集本身固定（它们是运行配置），但渲染模型的配置被记录以便复现渲染文本。

### 3.3 运行清单

每次实验运行产生一个 `run_manifest.json`，固定：
- 基准代码的 git SHA。
- 配置哈希（模型名称、预算、赛道、权重）。
- 数据集校验和（来自冻结清单）。
- 智能体模型修订版本、判定器模型修订版本（按提交哈希固定的 `lordx64/Qwable-v1`）。
- 运行时间戳和环境。

这使得任何报告结果都可以精确复现。

## 4. 数据集卡片

每个任务族的 `dataset_card.md` 是按照标准模板的结构化、可读摘要：

### 模板

```markdown
# 数据集：<family_name> Pilot (v1)

## 概览
- **族**：Research / Software-Dev
- **Episodes**：<n>
- **总会话数**：<n>
- **种子**：<list>
- **冻结日期**：<ISO 8601>
- **生成器版本**：<git SHA>

## 内容描述
<一段关于 episode 包含内容的摘要>

## 探针组成
| 类型 | 数量 | 评分方法 |
|------|------|---------|
| 事实回忆 | <n> | 程序化（ResearchChecker） |
| 实现 | <n> | 程序化（pytest 通过 SoftwareChecker） |
| 综合（开放式） | <n> | 判定器（`lordx64/Qwable-v1`） |
| 约束/行为 | <n> | 程序化（漂移不变量） |

## 范围合规
- [ ] 所有实体名称合成（无真实论文/作者）
- [ ] 无需网络访问
- [ ] 文件数 ≤ 6（SW-Dev）/ 不适用（Research）
- [ ] 所有 episode 通过 `validate_render`
- [ ] 所有 episode 通过 `lint_no_real_entities`

## 可复现性
- **SHA-256**（episodes.jsonl）：`<hash>`
- **SHA-256**（rendered/）：`<hash>`
- **重新生成验证**：<date> — 哈希一致
- **运行**：`python -m lhmsb.datasets regen-check --frozen datasets/<name>`

## 预期用途
对 <family> 任务上的长周期记忆系统进行基准测试。
不适用于训练或微调。

## 局限性
- 仅合成内容——不反映真实世界 <domain> 复杂性。
- 固定证据图（v1）——智能体动作不影响世界。
- Pilot 规模（<n> 个 episode）——不足以进行精细的系统排名。
```

## 5. 数据集生命周期

```
generate  ───→  validate  ───→  freeze  ───→  verify  ───→  regen-check
  │               │               │              │               │
  │  (Task 20)    │  (sim core    │  (write      │  (integrity   │  (prove
  │               │   + families) │   manifest)  │   check)      │   reproducible)
  ▼               ▼               ▼              ▼               ▼
episodes       render ok?      datasets/       checksums       hashes match
+ rendered     no leakage      <name>/         match           frozen set
text           no real ents    MANIFEST.json
                               dataset_card.md
```

所有步骤通过 `python -m lhmsb.datasets` 自动化。无人工策划，不手动编辑生成的 episode。
